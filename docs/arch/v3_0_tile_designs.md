# Tile 0: 5-Transistor OTA (Class A -- Analog Core)

## 1. Circuit Description

A classic single-ended-output operational transconductance amplifier (OTA). This is the most fundamental analog building block -- it converts a differential voltage into a proportional output current. The 5-transistor topology is the simplest practical OTA and the starting point for understanding all amplifier design.

### ASCII Schematic

```
                        VDD_A (1.8V)
                          |
               ┌──────────┼──────────┐
               |          |          |
            ┌──┴──┐    ┌──┴──┐      |
            | M3  |    | M4  |      |
            | PMOS|    | PMOS|      |
            | W/L:|    | W/L:|      |
            | 10/.5|   | 10/.5|     |
            └──┬──┘    └──┬──┘      |
               |  diode    |         |
               |  conn.    |         |
               ├───────────┤         |
               |           |         |
               |      out_main ──────┼──► OUT_MAIN
               |           |         |
            ┌──┴──┐    ┌──┴──┐      |
            | M1  |    | M2  |      |
            | NMOS|    | NMOS|      |
            | W/L:|    | W/L:|      |
            | 5/1 |    | 5/1 |      |
            └──┬──┘    └──┬──┘      |
               |          |         |
    stim_a ──►gate     gate◄── stim_b
    (V+)                       (V-)
               |          |
               └────┬─────┘
                    |
                 ┌──┴──┐
                 | M5  |
                 | NMOS|
                 | W/L:|
                 | 5/2 |
                 └──┬──┘
                    |
              tap1 ─┤──► TAP1 (tail current observation)
                    |
                   gate ◄── bias_code[9:0] (from current DAC)
                    |
                  VSS_A

    tap2 = Hi-Z (unused)

    LOAD SELECT (tile_param[11:10]):
      00 = High-Z (transconductor mode, no internal load)
      01 = 1K poly resistor to VSS_A
      10 = 10K poly resistor to VSS_A
      11 = 100K poly resistor to VSS_A

      Loads implemented as switchable poly resistors:

        out_main ──┬──[SW0]──[1K]──┬── VSS_A
                   ├──[SW1]──[10K]─┤
                   └──[SW2]──[100K]┘
```

### How It Works

- M1/M2 form the differential input pair. A positive differential voltage (V+ > V-) steers more tail current through M1, mirrored by M3/M4 to produce a rising output.
- M3/M4 form an active current mirror load, converting the differential current to a single-ended output.
- M5 sets the tail bias current. Its gate voltage is generated by a 10-bit binary-weighted current DAC controlled by `tile_param[9:0]`. This allows programmable gm.
- The output load is selectable to allow both transconductance-mode (high-Z output, measure current via external load) and voltage-gain-mode (internal resistive load) operation.

### Current DAC Implementation

The 10-bit current DAC is a binary-weighted array of unit NMOS current sources (sky130_fd_pr__nfet_01v8, W=1u/L=2u) whose gates share a common bias voltage generated by a diode-connected reference transistor. Each bit switches in 2^N unit sources:

```
  bias_code[0] → 1 unit   (100 nA)
  bias_code[1] → 2 units  (200 nA)
  bias_code[2] → 4 units  (400 nA)
  ...
  bias_code[9] → 512 units (51.2 uA)
  Total:         1023 units max (102.3 uA)
```

Unit current source: NMOS W=1u/L=2u, Vgs - Vth ~ 200mV, Id ~ 100nA.

## 8. Self-Test Pass/Fail Criteria (Mode 001)

| Check | Pass Condition | Error Code if Fail |
|-------|---------------|-------------------|
| Output within valid range | 0.3V < V_out < 1.5V at balanced input | TILE_ERR_RANGE (0x2) |
| Offset voltage | \|V_offset\| < 50 mV (generous bound for 130nm) | TILE_ERR_RANGE (0x2) |
| Output settles | Output stable within 10 mV over 32 consecutive cycles | TILE_ERR_UNSTABLE (0x3) |
| Bias current flowing | tap1 voltage > 100 mV (confirms M5 is conducting) | TILE_ERR_NO_STIMULUS (0x5) |
| Completion | DUT completes within timeout | TILE_ERR_TIMEOUT (0x1) |

**Self-test sequence (automated by DUT FSM):**
1. Set bias_code = 512 (mid-range, ~51.2 uA tail current)
2. Set load_sel = 10 (10K load)
3. Short stim_a = stim_b internally (via internal switch)
4. Wait 256 cycles for settling
5. Capture output voltage via internal comparator ladder (4-bit resolution)
6. Compare to expected mid-rail (VDD_A/2 = 0.9V, +/- 50mV)
7. Report offset = V_out - 0.9V in tile_result
8. Assert pass if all checks met, else assert error with appropriate code

## 10. Experiment Ideas (What Students Can Learn)

| Experiment | Setup | Learning Objective |
|------------|-------|-------------------|
| **DC operating point** | Mode 000. Sweep bias_code 0-1023, measure V_out at each. | Understand how bias current sets quiescent point. See saturation vs. triode. |
| **gm vs. bias current** | Mode 010. Fix V_in step size, sweep bias_code. Plot gain vs. I_tail. | The square-root law: gm ~ sqrt(2 * mu * Cox * W/L * Id). |
| **Input offset** | Mode 001. Run self-test on multiple chips. Histogram offsets. | Transistor mismatch statistics (Pelgrom model in 130nm). |
| **Gain vs. load resistance** | Mode 010. Fix bias_code = 512. Try load = 1K, 10K, 100K, Hi-Z. | Av = -gm * (Rout \|\| R_load). See gain drop with lower load. |
| **Common-mode range** | Mode 000. Sweep stim_a = stim_b from 0 to VDD. Find valid range. | ICMR limits from tail source saturation and load headroom. |
| **Frequency response** (advanced) | Route ring oscillator (tile 2) through DAC to stim_a. Measure output amplitude vs. frequency. | Bandwidth, dominant pole, GBW product. |
| **Tile-to-tile chaining** | Route tile 0 out_main to tile 3 (delay line) stim_a. | Multi-stage signal processing, analog signal chain concept. |
| **Power vs. performance** | Sweep bias_code from 1 to 1023. Measure gain, bandwidth, power at each. | The fundamental analog tradeoff: speed costs power. |

# Tile 2: Current-Starved Ring Oscillator (Class B -- Timing)

## 1. Circuit Description

A voltage-controlled oscillator (VCO) using a ring of current-starved inverters. The oscillation frequency depends on the starving current, which is controlled by an external voltage (Vctrl via stim_a) and a digital trim code. This tile generates its own clock -- it is the clock source, not a consumer.

### ASCII Schematic

```
                            VDD_A (1.8V)
                              |
            ┌─────────────────┼──────────────────────┐
            |                 |                      |
         ┌──┴──┐           ┌──┴──┐               ┌──┴──┐
         |M_P1 |           |M_P2 |      ...      |M_P5 |
         |PMOS |           |PMOS |               |PMOS |
         |starvP|          |starvP|              |starvP|    P-starve
         |W=2/.5|          |W=2/.5|              |W=2/.5|    transistors
         └──┬──┘           └──┬──┘               └──┬──┘
            |                 |                      |
         ┌──┴──┐           ┌──┴──┐               ┌──┴──┐
         |INV1 |           |INV2 |      ...      |INV5 |    Core
         |     |───node1───|     |───node2──...──|     |──┐ inverters
         └──┬──┘           └──┬──┘               └──┬──┘  |
            |                 |                      |     |
         ┌──┴──┐           ┌──┴──┐               ┌──┴──┐  |
         |M_N1 |           |M_N2 |      ...      |M_N5 |  | N-starve
         |NMOS |           |NMOS |               |NMOS |  | transistors
         |starvN|          |starvN|              |starvN|  |
         |W=1/.5|          |W=1/.5|              |W=1/.5|  |
         └──┬──┘           └──┬──┘               └──┬──┘  |
            |                 |                      |     |
            └─────────────────┼──────────────────────┘     |
                              |                            |
                            VSS_A                          |
                                                           |
         ┌─────────────────────────────────────────────────┘
         | feedback (odd number of stages = oscillation)
         |
         ▼
      INV1 input ◄──────────────────────────── INV5 output
                                                   |
                                              ┌────┴────┐
                                              | BUFFER  |
                                              | (2x inv)|──► out_main
                                              └─────────┘      (digital-level)

    STARVE CURRENT CONTROL:

      stim_a (Vctrl) ──► Bias generator ──► M_P* gates (PMOS starve)
                                         └► M_N* gates (NMOS starve)

      tile_param[9:0] ──► Trim current DAC ──► Added to/subtracted from
                           (fine adjustment)    starve current

    STAGE SELECT (tile_param[15:10]):
      Only 3 valid configurations:
        00_0001 = 1 stage  (3 inverters bypassed, 1+buffer active)
                  Note: 1-stage doesn't oscillate; used for delay measurement
        00_0011 = 3 stages (2 inverters bypassed)
        00_0101 = 5 stages (all active, default)

    INTERNAL TAP:

      node2 (between INV2 and INV3) ──► tap1 (analog, for jitter observation)
      tap2 = Hi-Z (unused)

    FREQUENCY COUNTER (internal, 16-bit):

      out_main ──► counter ──► tile_result[15:0] (frequency measurement)
      clk (system) ──► gate timer (programmable measurement window)
```

### How It Works

- A ring of an odd number of inverters oscillates because the signal inverts with each stage and the odd total creates instability (no stable DC point).
- Current-starving adds series transistors (M_P in pull-up, M_N in pull-down) that limit the charging/discharging current of each stage, slowing the transition and lowering the frequency.
- Vctrl (from stim_a) sets the bias voltage on the starve transistors. Higher Vctrl on M_N = more current = higher frequency. The digital trim provides fine adjustment.
- The output buffer ensures a clean digital-level signal at out_main even when the ring amplitude is reduced at low starving current.

### Frequency Calculation

```
f_osc = edge_count / (2 * gate_time_cycles * T_sysclk)

Example: edge_count = 500, gate_time = 1 (256 cycles), T_sysclk = 100ns (10 MHz)
  f_osc = 500 / (2 * 256 * 100ns) = 500 / 51.2us = 9.77 MHz
```

Division by 2 because each full cycle has 2 edges (rising + falling).

## Phase Noise Considerations

Ring oscillators have inherently poor phase noise compared to LC oscillators because:
1. **No energy storage** -- each cycle is generated from noise-contaminated transitions
2. **Every stage contributes** -- N stages means N independent noise sources
3. **Flicker noise dominates** at low offsets due to MOSFET 1/f noise in 130nm

Expected phase noise profile:
```
  Offset (Hz)    Phase Noise (dBc/Hz)    Region
  1 kHz          ~-40                     1/f^3 (flicker upconversion)
  10 kHz         ~-60                     1/f^3 to 1/f^2 transition
  100 kHz        ~-70                     1/f^2 (thermal noise)
  1 MHz          ~-80                     1/f^2
  10 MHz         ~-90                     Approaching noise floor
```

This is actually a useful teaching point: students can observe why LC oscillators exist (they trade area for phase noise performance).

# Tile 5: LDO-Lite (Class D -- Power)

## 1. Circuit Description

A minimal low-dropout linear voltage regulator. This is the simplest practical LDO: a PMOS pass transistor driven by an NMOS common-source error amplifier with resistive feedback. It regulates an output voltage by comparing a divided-down version of Vout to a reference voltage, then adjusting the pass transistor gate to maintain regulation.

This tile has special restrictions as a Class D (Power) tile: current limiting is mandatory, and only one Class D tile is allowed per chip to protect VDD_A.

### How It Works

1. **Reference input**: stim_a provides Vref (from the shared reference ladder, DAC, or external source).
2. **Error amplifier**: M2/M3 compare Vref to the divided-down Vout (fb_node). If Vout drops, fb_node drops, M3 conducts less, M2 drain rises, pulling M1 gate higher -- but M1 is PMOS, so higher Vgs means *less* current. Wait -- the topology is inverted:
   - Actually: if Vout drops, fb_node drops, M3 current decreases, error amp output (M4/M2 junction) rises, M1 gate voltage rises, |Vgs_M1| decreases... This is the wrong polarity for regulation.
   - Correction: The error amplifier uses M2 (Vref) as a *cascode* current source and M3 (fb) as the amplifying device with M4 load. When fb_node drops, M3 conducts less, M3 drain (= M1 gate) is pulled toward VDD by M4, turning M1 OFF more -- wrong again.
   - **Correct topology**: M2 has Vref on gate, M3 has fb_node on gate. They share a tail current source (M5, biased). Error amp output is M2's drain, which drives M1 gate. When Vout drops: fb_node drops, M3 conducts less of the tail current, more flows through M2, M2 drain drops, M1 gate drops, |Vgs_M1| increases, M1 passes more current, Vout rises. Negative feedback achieved.

   Simplified: the error amplifier is a differential pair (M2/M3) with a tail source (M5), and the output taken from M2's drain drives the PMOS pass gate.

3. **Pass transistor**: M1 is a wide PMOS. Its low on-resistance gives low dropout voltage. W=100um for < 200mV dropout at 1mA.
4. **Feedback divider**: Programmable R_top/R_bot sets the regulation voltage.
5. **Current limit**: A small sense resistor + comparator detects overcurrent and asserts an error flag, which can trigger M1 shutdown.
6. **Test load**: An internal ~10K resistor (enabled by tile_param[16]) provides a known load for self-test without requiring external routing.

### Current Limit Detail

The current sense resistor (10 ohm poly) develops a voltage proportional to load current. A simple comparator trips when V_sense > V_threshold, where V_threshold is set by the 6-bit ilimit_code driving a resistor ladder:

```
  V_threshold = (ilimit_code + 1) * 10 mV
  I_limit = V_threshold / R_sense = (ilimit_code + 1) * 10mV / 10ohm
          = (ilimit_code + 1) * 1 mA

  Wait -- that's too coarse. Corrected:
  R_sense = 100 ohm (higher for better resolution at low currents)
  V_threshold = (ilimit_code + 1) * 10 mV
  I_limit = V_threshold / R_sense = (ilimit_code + 1) * 100 uA
```

At ilimit_code = 9 (default): I_limit = 1 mA. At code 63: I_limit = 6.4 mA.

| **Current limiting** | Mode 000. Set low ilimit_code (e.g., 0 = 100uA limit). Enable test load (draws ~120uA). Observe overcurrent trip. | Overcurrent protection. Why current limits exist. |
| **Transient response** | Mode 001 (or manual). Toggle test load rapidly. Route tap1 to ADC. Observe error amp response. | Control loop dynamics, settling time, overshoot. |
| **Efficiency** | Mode 000. Measure I_in (via sense resistor on VDD_A side, if available) and I_out. Compute eta = (Vout * Iout) / (VDD * Iin). | Linear regulator efficiency = Vout/Vin. Why switching regulators exist. |
| **Noise observation** | Mode 000. Route out_main to ADC. Sample Vout at system clock rate. Compute spectrum in software. | LDO output noise, effect of loop bandwidth on noise filtering. |
| **Stability analysis** (advanced) | Route OTA tile (tile 0) output to LDO stim_a. Inject small-signal perturbation. Measure phase margin from step response ringing. | Loop stability, phase margin, pole-zero compensation concepts. |
| **Power supply for other tiles** | Route tile 5 out_main to tile 0 stim_a as a clean regulated supply. Compare OTA performance with regulated vs. raw VDD. | Real-world power supply application. Regulated supply improves analog performance. |

## Safety Considerations (Class D)

| Risk | Mitigation | Implementation |
|------|-----------|---------------|
| Excessive current draw damages VDD_A | Mandatory current limit (cannot be disabled, only adjusted) | Hardware comparator; limit code 0 still enforces 100uA minimum limit |
| Thermal damage to pass transistor | Max continuous current 6.4mA at 1.8V = 11.5 mW (well within package limits) | Current limit + thermal mass of die |
| Feedback divider misconfiguration | divider_code = 1023 with Vref = 0.9V requests Vout = 1.8V (= VDD); LDO enters dropout gracefully | Error amp saturates, M1 fully on, Vout = VDD - I*Rdson; no damage |
| Oscillation due to insufficient phase margin | Internal MIM decoupling cap (~2 pF); output capacitance from route matrix parasitics helps | If oscillation detected (TILE_ERR_UNSTABLE), tile auto-isolates |
| Startup latchup | Startup circuit ensures error amp reaches correct operating point before M1 turns on | 4-transistor startup sequence with enable gating |

## Open Questions (Draft)

1. **Internal reference for self-test**: Each tile's self-test assumes access to an internal reference voltage (~0.9V for OTA, ~0.6V for LDO). Should this come from the shared reference ladder via the route matrix, or should each tile have a crude local reference (e.g., diode-connected NMOS threshold)?

2. **ADC sharing during characterization**: Modes 010 (characterization) for all three tiles assume the host can read tile_result for digital measurement. The internal measurement uses a crude 4-bit comparator ladder (for self-test pass/fail). For precise characterization, the host should route tile outputs to the shared 12-bit ADC via the route matrix. This two-tier approach (crude internal + precise external) needs to be documented in the experiment guide.

3. **LDO output capacitance**: The LDO loop stability depends on output capacitance. The route matrix parasitics provide some (estimated 5-10 pF), and the internal MIM cap adds ~2 pF. If this is insufficient for stability, an external cap pad may be needed. Stability simulation required.

4. **Ring oscillator at high frequency**: At 100+ MHz, the ring oscillator output may couple into adjacent tiles through substrate. Guard ring spacing between T2 and analog tiles (T0, T1) should be maximized in the floorplan. Consider placing T2/T3 (timing tiles) on the opposite end of the tile array from T0/T1 (analog core tiles).

5. **Current DAC accuracy**: The binary-weighted current DAC in T0 and T2 may suffer from mismatch at the MSB transitions. For a learning platform this is acceptable (and educational), but students should be warned that DNL/INL errors exist. A future revision could use segmented architecture for the top 3-4 bits.
